# ISAP — Руководство пользователя

## Быстрый старт

### 1. Запуск PostgreSQL

```bash
# Если используете Docker:
docker-compose up db -d

# Или убедитесь, что PostgreSQL запущен локально на порту 5432
```

### 2. Применение миграций

```bash
cd backend
alembic upgrade head
```

### 3. Загрузка нормативов

```bash
python -m scripts.load_regulatory_data
```

### 4. Запуск бэкенда

```bash
python -m uvicorn src.main:app --reload --port 8000
```

### 5. Запуск фронтенда

```bash
cd ../frontend
npm run dev
```

### 6. Откройте в браузере

- **Фронтенд:** http://localhost:3000
- **Swagger API:** http://localhost:8000/docs

## Пошаговая инструкция

### Шаг 1: Создайте организацию

1. Перейдите в раздел **«Организации»** в боковом меню
2. Нажмите **«+ Добавить»** (или заполните форму сверху)
3. Заполните поля:
   - **Наименование** (обязательно) — например, «ООО Газпромнефть-Хантос»
   - **ИНН** (обязательно) — например, «8602060755»
   - Адрес, телефон, email — по желанию
4. Нажмите **«Создать»**

### Шаг 2: Добавьте ответственных лиц

1. Перейдите в раздел **«Ответственные лица»**
2. Нажмите **«+ Добавить»**
3. Заполните:
   - **Организация** — выберите из списка
   - **ФИО** (обязательно) — например, «Иванов И.И.»
   - Должность — «Начальник службы промышленной безопасности»
   - Роль — выберите «Начальник ПБ»
   - Телефон — например, «+7 (3452) 53-53-10»
4. Нажмите **«Создать»**
5. Повторите для других лиц (директор и т.д.)

### Шаг 3: Создайте объект ОПО

1. Перейдите в раздел **«Объекты (ОПО)»**
2. Через API (Swagger) создайте объект:
   ```bash
   curl -X POST http://localhost:8000/api/v1/facilities/ \
     -H "Content-Type: application/json" \
     -d '{
       "organization_id": "<UUID_организации>",
       "name": "Компрессорная станция Приобье",
       "facility_type": "Компрессорная станция",
       "hazard_class": 3,
       "reg_number": "РВ-86-00123",
       "address": "Ханты-Мансийский АО"
     }'
   ```

### Шаг 4: Добавьте оборудование

1. Перейдите в карточку объекта (нажмите «Детали»)
2. Выберите таб **«Оборудование»**
3. Нажмите **«+ Добавить»**
4. Заполните:
   - Наименование — «Компрессор К-500»
   - Тип — «Осевой компрессор»
   - Зав. номер — «SN-001»
5. Нажмите **«Добавить»**
6. Повторите для другого оборудования

### Шаг 5: Добавьте опасные вещества

1. В карточке объекта выберите таб **«Вещества»**
2. Нажмите **«+ Добавить»**
3. Заполните:
   - Наименование — «Природный газ (метан)»
   - CAS номер — «74-82-8»
   - Количество, кг — «50000»
4. Нажмите **«Добавить»**

### Шаг 6: Сгенерируйте ПМЛА

1. Перейдите в раздел **«Генерация ПМЛА»**
2. Выберите **Организацию** из выпадающего списка
3. Выберите **Объект ОПО**
4. Нажмите **«Сгенерировать ПМЛА»**
5. Дождитесь статуса **«На ревью»** (обычно 10-30 секунд)
6. Проверьте замечания (если есть)
7. Нажмите **«Утвердить»**

### Шаг 7: Скачайте документ

После утверждения появятся кнопки:
- **«Скачать DOCX»** — Word-документ
- **«Скачать PDF»** — PDF-версия

## API — Примеры запросов

### Через Swagger

Откройте http://localhost:8000/docs и используйте интерактивный UI.

### Через curl

**Создание организации:**
```bash
curl -X POST http://localhost:8000/api/v1/organizations/ \
  -H "Content-Type: application/json" \
  -d '{"name": "ООО Тест", "inn": "1234567890"}'
```

**Список организаций:**
```bash
curl http://localhost:8000/api/v1/organizations/
```

**Генерация ПМЛА:**
```bash
curl -X POST http://localhost:8000/api/v1/pmla/generate \
  -H "Content-Type: application/json" \
  -d '{
    "facility_id": "<UUID_объекта>",
    "context": {
      "organization": {"name": "ООО Тест", "inn": "1234567890"},
      "facility": {"name": "КС Приобье", "hazard_class": 3},
      "equipment": [{"name": "Компрессор"}],
      "substances": [{"name": "Метан", "quantity_kg": 50000}],
      "responsible_persons": [{"full_name": "Иванов", "phone": "+79991234567"}]
    }
  }'
```

**Скачивание DOCX:**
```bash
curl -O http://localhost:8000/api/v1/pmla/<UUID_документа>/download
```

**Скачивание PDF:**
```bash
curl -O http://localhost:8000/api/v1/pmla/<UUID_документа>/download/pdf
```

## Решение проблем

### Ошибка «Организация не найдена»

Убедитесь, что UUID организации передан верно. Список:
```bash
curl http://localhost:8000/api/v1/organizations/
```

### Ошибка «Документ не найден»

Документ создаётся при генерации. Проверьте статус:
```bash
curl http://localhost:8000/api/v1/pmla/<UUID>/status
```

### PDF не скачивается

PDF-конвертация требует MS Word (docx2pdf) или работает через fpdf2 fallback (без форматирования). Проверьте логи бэкенда.

### LLM не отвечает

Убедитесь, что Ollama запущен:
```bash
ollama list
ollama pull qwen2.5-coder:7b
```
