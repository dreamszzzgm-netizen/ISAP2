# Smart DOCX Import

DOCX-импорт нужен, чтобы пользователь мог загрузить старый ПМЛА или Word-опросник, а система попыталась предварительно заполнить анкету ПМЛА.

## Поддерживаемый режим

DOCX сейчас обрабатывается как безопасный preview.

```text
DOCX
  ↓
извлечение текста и таблиц
  ↓
распознавание ключевых полей
  ↓
одна строка preview в профиле pmla_questionnaire
  ↓
пользователь проверяет
  ↓
confirm import
  ↓
создаётся PmlaQuestionnaireModel
```

## Что извлекается

- организация;
- наименование ОПО;
- регистрационный номер ОПО;
- сведения об авариях/инцидентах;
- режим работы;
- численность персонала;
- сценарии аварий;
- силы и средства;
- ПАСФ;
- финансовый резерв;
- тренировки;
- приложения.

## API

DOCX используется через тот же Smart Import endpoint:

```bash
curl -X POST "http://localhost:8000/api/v1/imports/pmla_questionnaire/preview" \
  -H "Authorization: Bearer dev-secret" \
  -F "file=@old_pmla.docx"
```

После проверки preview:

```bash
curl -X POST "http://localhost:8000/api/v1/imports/jobs/{job_id}/confirm" \
  -H "Authorization: Bearer dev-secret"
```

## Ограничение

DOCX-импорт не должен автоматически создавать юридически значимые факты. Если система нашла упоминания аварий/инцидентов без явной фразы «не зарегистрированы», блок помечается как `требует проверки`.

Факты подтверждает пользователь.
