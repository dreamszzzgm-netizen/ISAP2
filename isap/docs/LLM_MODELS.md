# Выбор LLM-модели для ПМЛА

## Рекомендации

### Продакшен (облако)

| Модель | Провайдер | Качество русского | Стоимость | Рекомендация |
|--------|-----------|-------------------|-----------|--------------|
| `gemini-2.5-flash` | Google Gemini | 9/10 | Низкая | Рекомендуется для большинства задач |
| `gemini-2.5-pro` | Google Gemini | 9/10 | Средняя | Для сложных/критичных документов |
| `gpt-4o` | OpenAI | 8/10 | Средняя | Альтернатива |

**Настройка Gemini в `.env`:**
```bash
LLM_PROVIDER=openai
OPENAI_API_KEY=your-gemini-api-key
OPENAI_BASE_URL=https://generativelanguage.googleapis.com/v1beta/openai/
OPENAI_MODEL=gemini-2.5-flash
```

### Разработка (локально, Ollama)

| Модель | Параметры | Качество русского | VRAM | Рекомендация |
|--------|-----------|-------------------|------|--------------|
| `llama3:8b` | 8B | 7/10 | 6GB | Рекомендуется |
| `mistral-nemo` | 12B | 8/10 | 8GB | Альтернатива |
| `qwen2.5:14b` | 14B | 7/10 | 10GB | Альтернатива |
| `gemma2:27b` | 27B | 8/10 | 16GB+ | Если есть ресурсы |

**Установка модели:**
```bash
ollama pull llama3:8b
```

## Fallback-цепочка

Система автоматически переключается на запасной провайдер при ошибке:

1. **Основной:** Ollama (llama3:8b) — для разработки
2. **Запасной:** Gemini (gemini-2.5-flash) — для продакшена

Настройка в `.env`:
```bash
LLM_FALLBACK_ENABLED=true
```

## Тестирование моделей

Для сравнения качества генерации на разных моделях:

```bash
cd backend
python -m scripts.test_llm_models
```

Скрипт тестирует модели на одном и том же промпте и выводит:
- Время генерации
- Количество токенов
- Наличие утечки `[CALCULATED:...]` плейсхолдеров
- Наличие английских слов

## Почему не qwen2.5-coder:7b?

Текущая модель `qwen2.5-coder:7b` оптимизирована для генерации кода:
- Обучена преимущественно на английском техническом тексте
- Генерирует «кальки» с английского вместо русских формулировок
- Не знает нормативных формулировок НПА

Для ПМЛА нужна модель, обученная на русскоязычном профессиональном тексте.
