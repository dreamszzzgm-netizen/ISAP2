# PMLA — Engineer Review Checklist (DOCX)

**Дата:** 2026-07-09
**Стадия:** PMLA MVP — First Real/Anonymized OPO validation
**Назначение:** Чеклист инженерной проверки сгенерированного DOCX перед постановкой статуса `approved`. Без прохождения этого чеклиста статус `approved` не ставится.

> Этот чеклист проверяет **готовый DOCX**, а не анкету. Анкета проверяется через quality review (см. `PMLA_REAL_OPO_VALIDATION_WALKTHROUGH.md`, раздел 8).

---

## Как пользоваться

- Откройте скачанный DOCX (`PMLA_<CODE>.docx`).
- Пройдите каждый раздел по порядку.
- В колонке «Результат» ставьте: `OK` / `Замечание` / `Н/П` (неприменимо).
- Все `Замечание` переносите в `PMLA_REAL_OPO_<CODE>_FINDINGS.md` с типом (`содержание` / `формат` / `структура` / `нормативное`).
- Финальный вердикт в конце чеклиста.

**Вердикт по пригодности DOCX как инженерного черновика:**

- [ ] **Пригоден** — замечания несущественны, допустимо `approved` после их учёта.
- [ ] **Пригоден с доработкой** — требует `needs_changes`, устранение замечаний, затем повторная проверка.
- [ ] **Непригоден** — критические дефекты; регенерация или доработка данных обязательна.

**Статус в review workflow (выбрать один из `allowed_transitions`):**

- [ ] `needs_changes` (есть замечания)
- [ ] `approved` (проверен, пригоден как черновик)
- [ ] `ready_to_issue` (подготовлен к выдаче — по согласованию)
