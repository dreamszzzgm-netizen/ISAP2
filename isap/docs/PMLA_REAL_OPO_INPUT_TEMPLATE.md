# PMLA — First Real/Anonymized OPO: Input Data Template

**Дата:** 2026-07-09
**Стадия:** PMLA MVP — First Real/Anonymized OPO validation
**Назначение:** Единый шаблон для сбора исходных данных по первому реальному (или обезличенному) ОПО перед заполнением анкеты ПМЛА и генерацией DOCX.

> ⚠️ Это **шаблон сбора данных**, а не автопарсер. Заполнять вручную. Поля сопоставлены с блоками анкеты ПМЛА (`backend/data/demo_pmla_validation.json` → `questionnaire.data`) и моделями справочников, чтобы данные переносились в систему 1:1.
>
> 🔒 Все реальные данные **обезличиваются** до занесения в систему (см. `PMLA_REAL_OPO_VALIDATION_WALKTHROUGH.md`, раздел «Обезличивание»).

---

## Как пользоваться этим шаблоном

1. Скопируйте этот файл в `docs/inbox/PMLA_REAL_OPO_<CODE>.md` (например `<CODE>` = `OPO-GAS-01`).
2. Заполняйте только поля, для которых есть реальные (обезличенные) данные.
3. Неизвестные поля **не выдумывайте** — пометьте в разделе «Поля, требующие уточнения».
4. Текстовые поля вводите в том виде, в каком они должны попасть в DOCX.
5. Числовые поля — без единиц измерения в самом значении (единицы указаны в подсказке).

Условные обозначения:
- `[обязательно]` — без этого поля генерация невозможна или quality review вернёт `critical`.
- `[рекомендуется]` — влияет на score quality review.
- `[опционально]` — улучшает полноту, но не блокирует.

## Контроль перед передачей в систему

Перед тем как переносить данные в анкету ПМЛА, убедитесь:

- [ ] Все обязательные поля (`[обязательно]`) заполнены или помечены как «требует уточнения».
- [ ] Данные обезличены (нет реальных ИНН, ФИО, точных адресов, если ОПО обезличенный).
- [ ] Числовые значения — числа без текста единиц измерения в значении.
- [ ] Сценарии аварий либо выбраны из матрицы, либо описаны как кастомные.
- [ ] Раздел «Поля, требующие уточнения» заполнен для каждого пробела.

> После заполнения — следуйте `PMLA_REAL_OPO_VALIDATION_WALKTHROUGH.md`.
