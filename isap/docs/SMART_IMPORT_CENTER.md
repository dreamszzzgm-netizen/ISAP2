# Smart Import Center

## Назначение

Smart Import Center — единый механизм импорта данных из Excel/CSV с предпросмотром, сопоставлением колонок, проверкой обязательных полей, поиском дублей и подтверждением перед записью в БД.

Главное правило: импорт никогда не пишет данные в базу сразу после загрузки файла.

```text
Upload → Parse → Map Columns → Normalize → Validate → Duplicate Check → Preview → Confirm → DB → Report
```

## Первый набор профилей

### `fire_departments`

Импорт пожарных подразделений в справочник `emergency_services`.

Минимальные поля:

- Наименование
- Адрес

Дополнительные поля:

- Телефон
- Телефон диспетчера
- Муниципальный район
- Населённый пункт
- Координаты
- Зона обслуживания
- Примечание

### `pasf_units`

Импорт ПАСФ / АСФ в справочник `emergency_rescue_units`.

Минимальные поля:

- Наименование
- Номер свидетельства

Дополнительные поля:

- Юридический адрес
- Фактический адрес / место дислокации
- Телефон диспетчера
- Email
- Руководитель
- Дата свидетельства
- Срок действия свидетельства
- Виды работ
- Паспорт / оснащение
- Количество спасателей
- Режим готовности
- Район обслуживания

### `pmla_questionnaire`

Импорт анкеты ПМЛА в `pmla_questionnaires`.

Минимальные поля:

- Организация
- ОПО

Поддерживаемые блоки:

- аварии / инциденты
- режим работы
- численность персонала
- выбранные сценарии
- пользовательские сценарии / другое
- силы и средства
- ПАСФ
- финансовый резерв
- тренировки
- приложения

## API

### Список профилей

```http
GET /api/v1/imports/profiles
```

### Предпросмотр импорта

```bash
curl -X POST "http://localhost:8000/api/v1/imports/fire_departments/preview" \
  -H "Authorization: Bearer dev-secret" \
  -F "file=@fire_departments.xlsx"
```

Ответ содержит:

- `job.id`
- `header_mapping`
- первые 100 строк предпросмотра
- ошибки
- предупреждения
- возможные дубли

### Получить строки предпросмотра

```http
GET /api/v1/imports/jobs/{job_id}/rows
```

### Подтвердить импорт

```bash
curl -X POST "http://localhost:8000/api/v1/imports/jobs/{job_id}/confirm" \
  -H "Authorization: Bearer dev-secret"
```

## Как это использовать в ПМЛА

1. Импортировать ПАСФ / АСФ.
2. Импортировать пожарные подразделения и другие аварийные службы.
3. Импортировать или заполнить анкету ПМЛА.
4. В карточке генерации ПМЛА выбирать ПАСФ и службы не вручную, а из справочников.
5. Для сценариев аварий использовать авто-предложения + пункт `Другое` для ручного ввода.

## Где можно подключить LLM

LLM можно использовать только как помощника:

- распознать неизвестную колонку;
- разобрать строку оборудования;
- объяснить пользователю предупреждения;
- предложить недостающие силы и средства;
- сформулировать официальный текст для ПМЛА.

LLM не должна самостоятельно придумывать факты: телефоны, адреса, свидетельства, договоры, даты и суммы.


## DOCX import

Smart Import также поддерживает `.docx` для профиля `pmla_questionnaire`. DOCX превращается в preview-строку анкеты ПМЛА и требует подтверждения пользователя перед записью в базу. Подробнее: `docs/SMART_DOCX_IMPORT.md`.
